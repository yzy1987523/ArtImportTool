## 阶段说明

### 阶段1: 核心数据流验证（2-3天）
验证最小可行路径，确保核心数据模型正确。

- **1.1 数据库Schema验证** ✅
  - 创建数据库表和索引
  - 初始化默认数据
  - 编写Schema验证测试

- **1.2 资源入库流程验证** ✅
  - 文件哈希计算
  - 资源去重逻辑
  - 资源元数据提取
  - 批量导入性能测试

- **1.3 Tag系统验证** ✅
  - Tag CRUD操作
  - 资源Tag关联
  - 按Tag筛选资源
  - 多Tag组合查询

- **1.4 Project管理验证** ✅
  - Project创建
  - 资源与Project关联
  - Project资源查询
  - 资源所属Project查询

### 阶段2: Unity集成验证（3-4天）
验证Unity导入和路由表维护机制。

- **2.1 Unity编辑器窗口** ✅
  - 创建资源浏览窗口
  - 显示数据库中的资源列表
  - 支持搜索和筛选
  - 支持资源选择

- **2.2 资源导入Unity** ✅
  - 从数据库复制文件到Unity项目
  - 获取Unity资源GUID
  - 创建路由表记录
  - 关联到Project

- **2.3 路由表维护** ✅
  - 实现UnityRoutes表
  - 实现RouteHistory表
  - 路由更新和历史记录
  - 批量操作和回滚

- **2.4 风格化资源上传** ✅
  - 实现StyleMigrations表
  - 实现名称匹配算法
  - 创建风格迁移关联
  - Unity上传窗口

### 阶段3: 资源替换验证（2-3天）
验证完整的资源替换工作流。

- **3.1 资源替换功能** ✅
  - 查询资源的风格化版本
  - 更新路由表指向
  - 替换Unity中的资源引用
  - 记录替换历史

- **3.2 批量替换** ✅
  - 按Tag批量选择资源
  - 批量替换为指定风格
  - 支持回滚操作

- **3.3 WPF界面（可选）** ⏭️
  - 跳过（Unity扩展已提供完整功能）
- **3.4 性能和压力测试** ⏳
